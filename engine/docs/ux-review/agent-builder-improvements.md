# UX-ревью Agent Builder — План улучшений

**Дата:** 2026-04-03
**Scope:** Visual Agent Builder (админ-панель, роут `/builder`)
**Проверенные компоненты:** AgentBuilderPage, AgentNode, TriggerNode, BuilderSidePanel, BuilderChat, BuilderAssistant, BuilderFlowTest, BuilderExportImport, DriftNotification

---

## Критические (блокируют юзабилити или вызывают проблемы с данными)

### 1. YAML-вкладка показывает сохранённые данные, а не текущие правки
- **Проблема:** `agentToYaml(agent)` использует серверный проп, а не `form` state. Пользователь редактирует system prompt → переключается на YAML → видит СТАРЫЙ промпт. Кнопка Copy копирует устаревшие данные.
- **Решение:** Заменить на `agentToYaml(form)` или объединённое представление
- **Файлы:** `BuilderSidePanel.tsx:695`, `BuilderSidePanel.tsx:347`
- **Трудозатраты:** Малые

### 2. Ошибки API при connect/disconnect невидимы пользователю
- **Проблема:** Все catch-блоки используют только `console.error()`. Нет тостов, нет inline-ошибок. Путь `onEdgesDelete` особенно опасен: React Flow удаляет edge с canvas ДО вызова API — при ошибке API canvas и сервер рассинхронизируются.
- **Решение:** Добавить toast/notification компонент. Восстанавливать edge при ошибке API в `onEdgesDelete`. Показывать inline-ошибку в `confirmDelete`.
- **Файлы:** `AgentBuilderPage.tsx:353,385,425,558`
- **Трудозатраты:** Средние

### 3. Нет системы toast/notification в билдере
- **Проблема:** Нет универсального механизма уведомлений. Каждая ошибка API невидима. Только диалог создания агента показывает inline-ошибки.
- **Решение:** Добавить легковесный toast-компонент (или переиспользовать из проекта)
- **Файлы:** Новый компонент + интеграция в AgentBuilderPage
- **Трудозатраты:** Средние

### 4. Пустое состояние без кликабельной CTA-кнопки
- **Проблема:** "No agents yet / Click + Add Agent" имеет `pointer-events-none`. Пользователь должен искать кнопку в тулбаре. First-time experience полностью пассивный.
- **Решение:** Добавить inline-кнопку "Создать первого агента" в пустом состоянии. Опционально — иллюстрацию (два соединённых контура нод).
- **Файлы:** `AgentBuilderPage.tsx:653-658`
- **Трудозатраты:** Малые

### 5. Поле имени агента без real-time валидации
- **Проблема:** `pattern="^[a-z][a-z0-9-]*$"` — только HTML5, нет обёртки `<form>`, `handleCreate` проверяет только пустоту. Невалидные имена ("My Agent", пробелы, uppercase) проходят на сервер и отклоняются на round-trip.
- **Решение:** Валидация в `onChange`, авто-lowercase, inline красный текст. Trim ввода.
- **Файлы:** `AgentBuilderPage.tsx:793-801, 434`
- **Трудозатраты:** Малые

## Важные (существенно ухудшают опыт)

### 6. Хэндлы почти невидимы — низкая обнаруживаемость drag-to-connect
- **Проблема:** Круги 2.5x2.5 цвета `#87867F` сливаются с рамкой ноды. Видны только при наведении. Новые пользователи не могут найти точки подключения.
- **Решение:** Увеличить до 3x3, использовать более яркий цвет по умолчанию (`#A0A090`), рассмотреть кольцевой контур или always-visible состояние.
- **Файлы:** `AgentNode.tsx:36-40, 119-123`
- **Трудозатраты:** Малые

### 7. Нет визуальной обратной связи при перетаскивании соединения
- **Проблема:** Нет `connectionLineStyle`, `onConnectStart`, `onConnectEnd`. Нет подсветки валидных/невалидных целей. Пользователь тянет вслепую.
- **Решение:** Добавить `connectionLineStyle` (accent color), `isValidConnection`, подсветку совместимых хэндлов через `onConnectStart`.
- **Файлы:** `AgentBuilderPage.tsx:619-651`
- **Трудозатраты:** Средние

### 8. Невалидные соединения отклоняются молча
- **Проблема:** Самосоединение, trigger-to-agent, дубликат — всё молча игнорируется. Пользователь завершает drag, ничего не происходит, нет объяснения.
- **Решение:** Показать транзиентный toast: "Нельзя соединить агента с самим собой" / "Соединение уже существует" / "Соединения триггеров управляются на странице Triggers"
- **Файлы:** `AgentBuilderPage.tsx:323-358`
- **Трудозатраты:** Малые (при наличии toast-системы)

### 9. Нет контекстного меню при правом клике на ноду
- **Проблема:** Правый клик на ноду агента открывает контекстное меню canvas (Add Agent, Add Trigger) — контекстуально неправильно. Ожидается: Edit, Delete, Duplicate, Focus Connections.
- **Решение:** Добавить обработчик `onNodeContextMenu` с действиями, специфичными для ноды.
- **Файлы:** `AgentBuilderPage.tsx:619-651`
- **Трудозатраты:** Средние

### 10. Правый клик на edge триггера ничего не делает — нет объяснения
- **Проблема:** Правый клик на триггерный (фиолетовый) edge молча возвращает. Пользователь не знает, что они read-only.
- **Решение:** Показать read-only контекстное меню: "Управляется на странице Triggers" со ссылкой.
- **Файлы:** `AgentBuilderPage.tsx:531-532`
- **Трудозатраты:** Малые

### 11. Клик на ноду триггера ничего не делает
- **Проблема:** Клик визуально выделяет (фиолетовое свечение), но не открывает панель, нет деталей, нет действий. Сбивает с толку.
- **Решение:** Показать read-only панель деталей триггера или навигацию на страницу Triggers для редактирования.
- **Файлы:** `AgentBuilderPage.tsx:504-505`
- **Трудозатраты:** Средние

### 12. Клик на canvas закрывает боковую панель — возможна потеря несохранённых правок
- **Проблема:** `onPaneClick` безусловно закрывает боковую панель. Нет проверки dirty-state. Случайный клик = потеря правок.
- **Решение:** Если `isDirty`, показать подтверждение "Несохранённые изменения" перед закрытием. Или автосохранение.
- **Файлы:** `AgentBuilderPage.tsx:523-527`, `BuilderSidePanel.tsx`
- **Трудозатраты:** Средние

### 13. Нет защиты от потери несохранённых изменений при закрытии/навигации
- **Проблема:** Escape, клик вне панели, кнопка "Full Edit" — всё закрывает панель, молча отбрасывая dirty-правки.
- **Решение:** Добавить guard в стиле `beforeUnload` при `isDirty === true`.
- **Файлы:** `BuilderSidePanel.tsx`, `AgentBuilderPage.tsx:205-217`
- **Трудозатраты:** Средние

### 14. Нет поддержки Ctrl+Z (Undo)
- **Проблема:** Каждое соединение/разъединение — немедленный API-вызов. Случайное удаление edge необратимо без ручного воссоздания. Базовое ожидание для визуальных билдеров.
- **Решение:** Реализовать стек undo (как минимум для операций с edge). Сложно, но высокая ценность.
- **Файлы:** `AgentBuilderPage.tsx` (новое управление состоянием)
- **Трудозатраты:** Большие

### 15. fitView не срабатывает после auto-layout
- **Проблема:** `fitView` — проп (срабатывает только при mount). После пересчёта dagre через "Auto Layout" viewport не подстраивается. Ноды могут уйти за экран.
- **Решение:** Вызвать `reactFlowInstance.fitView()` после `setNodes(laid)`.
- **Файлы:** `AgentBuilderPage.tsx:403-409`
- **Трудозатраты:** Малые

### 16. Подписи на edge почти нечитаемы (контраст 2.5:1)
- **Проблема:** Текст `#87867F` на фоне `#1F1F1F` при 10px = не проходит WCAG AA. Подписи "spawns" и "triggers" очень трудно прочитать.
- **Решение:** Повысить до `#CBC9BC` (brand-shade2), увеличить до 11px.
- **Файлы:** `AgentBuilderPage.tsx:93-94, 107-108`
- **Трудозатраты:** Малые

### 17. "Add Trigger" доступен только через контекстное меню (ПКМ)
- **Проблема:** Нет кнопки в тулбаре. ПКМ не обнаруживается большинством пользователей. Hint bar упоминает ПКМ, но сам почти невидим.
- **Решение:** Добавить кнопку "Add Trigger" в тулбар (рядом с Add Agent, или как split button).
- **Файлы:** `AgentBuilderPage.tsx:570-600`
- **Трудозатраты:** Малые

### 18. Disabled trigger: `opacity-50` затемняет и бейдж "disabled"
- **Проблема:** При `enabled=false` вся нода на 50% прозрачности. Бейдж "disabled" становится невидимым (и так низкоконтрастный текст, уменьшенный вдвое).
- **Решение:** Применить прозрачность только к телу, бейдж оставить на полной непрозрачности. Или использовать другой паттерн (зачёркнутый заголовок, приглушённый overlay иконки).
- **Файлы:** `TriggerNode.tsx:44`
- **Трудозатраты:** Малые

### 19. Textarea system prompt — только 6 строк, без resize
- **Проблема:** Промпты часто 20-100+ строк. В панели 320px с 6 строками пользователь должен скроллить через крошечный viewport. `resize-none` блокирует расширение.
- **Решение:** Разрешить `resize-y` или добавить toggle expand/collapse. Минимум `rows={10}`.
- **Файлы:** `BuilderSidePanel.tsx:418`
- **Трудозатраты:** Малые

### 20. "Confirm Before" использует свободный текстовый ввод вместо picker инструментов
- **Проблема:** Пользователи должны набирать точные имена инструментов по памяти. Секция Tools уже имеет структурированный chip-based выбор с метаданными.
- **Решение:** Переиспользовать chip-based picker инструментов, отфильтрованный по уже выбранным.
- **Файлы:** `BuilderSidePanel.tsx:578-618`
- **Трудозатраты:** Средние

### 21. Отсутствует поле escalation в боковой панели
- **Проблема:** `AgentDetail` имеет `escalation`, но боковая панель не показывает и не редактирует его. Пользователи, настроившие через API, не видят это поле.
- **Решение:** Добавить сворачиваемую секцию Escalation (dropdown действия, webhook URL, список триггеров).
- **Файлы:** `BuilderSidePanel.tsx`
- **Трудозатраты:** Средние

### 22. BuilderChat не сбрасывает сессию при ошибке
- **Проблема:** В отличие от BuilderAssistant, который сбрасывает `sessionIdRef` при ошибке, BuilderChat сохраняет устаревший session_id. Последующие сообщения могут вызывать повторные ошибки.
- **Решение:** Добавить `sessionIdRef.current = ''` в пути ошибок (тот же паттерн, что в BuilderAssistant).
- **Файлы:** `BuilderChat.tsx:76-86`
- **Трудозатраты:** Малые

### 23. Chat vs AI Assistant — пересекающееся назначение, неясное различие
- **Проблема:** Вкладка Chat тестирует выбранного агента. AI Assistant настраивает платформу. Но ничто не объясняет это пользователю.
- **Решение:** Добавить empty-state пояснение во вкладку Chat: "Тестируйте ответы этого агента. Для настройки платформы используйте AI Assistant."
- **Файлы:** `BuilderChat.tsx:226`
- **Трудозатраты:** Малые

### 24. Тулбар не адаптивный — кнопки выходят за экран на узких экранах
- **Проблема:** Плоский `flex gap-3`, нет `flex-wrap`, нет breakpoints. Кнопки наезжают друг на друга при ширине менее ~900px.
- **Решение:** Добавить `flex-wrap` или свернуть вторичные действия в dropdown при breakpoint.
- **Файлы:** `AgentBuilderPage.tsx:570`
- **Трудозатраты:** Средние

### 25. Flow Test не поддерживает multi-turn сессии
- **Проблема:** Каждое тестовое сообщение stateless (`{ message: text }` без `session_id`). Нельзя тестировать multi-step agent flows.
- **Решение:** Добавить отслеживание сессии (как в BuilderAssistant), показывать историю разговора, добавить кнопку "New Session".
- **Файлы:** `BuilderFlowTest.tsx:90`
- **Трудозатраты:** Средние

### 26. Состояние формы не сбрасывается при повторном открытии диалогов
- **Проблема:** Диалоги New Agent и New Trigger сохраняют предыдущие значения после отмены. Очищаются только при успешном создании.
- **Решение:** Добавить `setNewForm(DEFAULT_NEW_FORM)` и `setTriggerForm({...})` при открытии диалога.
- **Файлы:** `AgentBuilderPage.tsx:595, 710`
- **Трудозатраты:** Малые

### 27. Dismiss drift-уведомления сбрасывается при каждом обновлении canvas
- **Проблема:** `setDismissed(false)` срабатывает при каждом изменении `checkTrigger`. Пользователь закрывает → AI Assistant отвечает → баннер появляется снова. "Whack-a-mole".
- **Решение:** Не сбрасывать `dismissed` при изменении `checkTrigger`, или сохранять в localStorage.
- **Файлы:** `DriftNotification.tsx:65-66`
- **Трудозатраты:** Малые

### 28. Drift-уведомление без кнопки "Ре-экспорт"
- **Проблема:** Баннер сообщает "changed since last export", но предлагает только dismiss. Пользователь должен искать кнопку Export в тулбаре.
- **Решение:** Добавить inline-кнопку "Export Now" в баннер drift.
- **Файлы:** `DriftNotification.tsx:71-90`
- **Трудозатраты:** Малые

### 29. Экспорт не показывает обратной связи об успехе
- **Проблема:** Файл скачивается молча. Нет тоста, нет подтверждения. Пользователь может сомневаться, сработало ли.
- **Решение:** Показать краткий toast "Конфигурация экспортирована" после скачивания.
- **Файлы:** `BuilderExportImport.tsx:15-38`
- **Трудозатраты:** Малые (при наличии toast-системы)

### 30. Несогласованные шорткаты отправки: Enter (Assistant) vs Ctrl+Enter (Flow Test)
- **Проблема:** Два текстовых поля на одной странице используют разные шорткаты отправки.
- **Решение:** Стандартизировать. Рекомендация: Enter для отправки везде, Shift+Enter для новой строки.
- **Файлы:** `BuilderAssistant.tsx:258`, `BuilderFlowTest.tsx:194`
- **Трудозатраты:** Малые

### 31. AI Assistant / Flow Test — наложение панелей
- **Проблема:** Обе могут быть открыты одновременно. Assistant — `fixed bottom-right z-60 w-96`, Flow Test — flex child `w-96` у правого края. Assistant накладывается на Flow Test.
- **Решение:** Перепозиционировать assistant при открытом Flow Test, или запретить одновременное отображение.
- **Файлы:** `BuilderAssistant.tsx:281`, `AgentBuilderPage.tsx:670-686`
- **Трудозатраты:** Средние

### 32. AI Assistant — фиксированная высота, нет resize
- **Проблема:** `h-[500px]` захардкожено. Слишком много на ноутбуках, тратит место на больших экранах. Нет минимизации.
- **Решение:** Добавить drag-to-resize или хотя бы toggle minimize/maximize.
- **Файлы:** `BuilderAssistant.tsx:281`
- **Трудозатраты:** Средние

### 33. Слишком агрессивное обрезание tool calls — нет расширения
- **Проблема:** Flow Test обрезает на 120 символов, Chat/Assistant на 80. Для отладки нужны полные JSON payload.
- **Решение:** Click-to-expand или copy-to-clipboard на блоках tool call.
- **Файлы:** `BuilderFlowTest.tsx:373`, `BuilderChat.tsx:245`, `BuilderAssistant.tsx:392`
- **Трудозатраты:** Малые

### 34. Cron-выражение без валидации и пресетов
- **Проблема:** Только placeholder (`0 9 * * *`) и текст-подсказка. Нет клиентской валидации. Типичные ошибки проходят на сервер.
- **Решение:** Добавить dropdown пресетов ("Каждый час", "Ежедневно в 9:00", "Еженедельно в понедельник") + базовая regex-валидация.
- **Файлы:** `AgentBuilderPage.tsx:889-899`
- **Трудозатраты:** Средние

### 35. Удаление агента показывает спиннер, но не текст ошибки при неудаче
- **Проблема:** Catch-блок `confirmDelete` только логирует в консоль. Диалог остаётся открытым с re-enabled кнопкой, но без сообщения об ошибке.
- **Решение:** Добавить `setDeleteError(err.message)` и показывать inline в ConfirmDialog.
- **Файлы:** `AgentBuilderPage.tsx:413-430`
- **Трудозатраты:** Малые
